P2A 로컬 산출물 저장 서버를 개발하고 싶다.

현재 P2A는 기획 문서, task graph, run 기록, proposal, review 같은 산출물을 로컬 md/json 파일로 저장한다. 이 파일 기반 구조는 유지하되, 별도의 저장 서버에 project / iteration / task / run / document 관계를 보존해서 동기화하고 싶다.

1차 목표는 RAG 완성이 아니라, 향후 RAG와 자기고도화를 위한 저장 기반을 만드는 것이다.

기술스택은 Kotlin + Spring Boot + PostgreSQL + pgvector를 사용한다. 서버는 REST API를 제공하고, P2A CLI/GUI가 나중에 연동할 수 있어야 한다.

MVP 범위:
- 프로젝트 등록
- iteration 등록
- 문서 스냅샷 저장
- task graph 저장
- task 저장
- run 기록 저장
- 문서 chunk 저장
- content hash 기반 중복 방지
- projectId / iterationId / taskId / runId / artifactType / sourcePath 기준 조회
- keyword 검색
- vector 검색을 위한 pgvector 스키마 준비
- embedding 값은 초기에는 외부에서 주입 가능한 형태로만 저장하고, 서버가 AI API를 직접 호출하지 않는다
- 모든 자동 개선 적용은 하지 않는다. 자기고도화는 후속 단계에서 proposal 생성까지만 고려한다.

중요한 제약:
- 로컬 파일이 원본이고, 저장 서버는 동기화/검색/회고용 보조 저장소다.
- 서버는 P2A 하네스를 직접 실행하지 않는다.
- 서버는 AI agent를 직접 실행하지 않는다.
- 초기에는 로컬 개발용 단일 사용자 기준으로 만든다.
- 인증은 단순 API key 또는 local token이면 충분하다.
- Docker Compose로 PostgreSQL + pgvector를 실행할 수 있어야 한다.
- 테스트는 Testcontainers 기반으로 작성한다.

이 요구사항을 기반으로 P2A intake/spec/task graph를 만들어줘.
