## 읽는 법

- rank 1-3, 7, 8은 **로컬 전용 내부 품질 수정**: 근거가 직접 관찰된 로컬 코드(위에서 `local-derived`로 relabel)이며, "외부" 노트는 비교 store가 그 capability를 표준으로 본다는 것뿐 — 독립적 시장 수요가 아니다. 품질/foundation 작업엔 적절하나, 외부 신호로 이중계상하지 않도록 명시한다.
- rank 4-6은 검색 품질 코어로, guardrail 의존성을 지키도록 배열: hybrid retrieval(r4, parity) → retrieval-eval 하네스(r5) → 고비용 ANN 스키마 재설계(r6, r5 없이 배포 금지). r4도 결과를 바꾸므로 이상적으로는 r5가 r4보다 먼저 안착.
- rank 9(citation)는 진짜 외부 leg를 가진 몇 안 되는 품질 항목 중 하나(provenance/citation 요청, S12).
- rank 10-11은 진짜 차별화 베팅. 설계 주도이고 수요 게이트: 근거가 개발자 도구 신호(mem0/cognee/nbdime에서 개발자가 반복 요청하는 것)이며 검증된 시장/구매 수요가 아니다. 구축 노력을 투입하기 전 실제 코딩 에이전트 팀과 검증할 것.

진짜 외부 신호 leg를 가진 rank: 4, 9, 10, 11 (및 1의 retrieval 품질 맥락). rank 1-3, 5, 7, 8은 주로 로컬 `path:line` 근거에 anchor되며, 외부 참조는 "표준 관행이다"이지 독립적 수요가 아니므로 그렇게 읽어야 한다. rank 10-11은 medium 신뢰도로 유지하고 수요 게이트로 표시 — 외부 근거가 개발자 신호일 뿐이기 때문. 시점 민감 외부 주장(Spring AI 2.0 GA 범위 2026-06 기준; cognee contradiction 처리 "진행 중")은 사용 전 재확인할 것.
